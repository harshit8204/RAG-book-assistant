## 🔍 What Is This?

**RAG Book Assistant** is an AI-powered document Q&A system. Upload any PDF — textbook, research paper, or manual — and ask natural language questions. Instead of relying on a generic LLM that may hallucinate, this system retrieves the exact relevant sections from your document and passes them as context to the LLM, making every answer grounded and traceable.

---

## ✨ Features

- **PDF Upload & Processing** — Upload any PDF directly through the browser; parsed and chunked automatically
- **MMR Retrieval** — Uses Maximal Marginal Relevance to fetch diverse, non-redundant relevant chunks
- **ChromaDB Vector Store** — Embeddings stored and queried via ChromaDB for fast semantic search
- **HuggingFace Embeddings** — Uses `sentence-transformers/all-MiniLM-L6-v2` locally — no embedding API cost
- **Mistral AI LLM** — Answers generated by `mistral-small-latest` using only document context
- **Retrieved Context Panel** — Expandable section shows the exact document chunks used to generate the answer
- **Multiple Retriever Implementations** — Includes similarity search, MMR, multi-query, and ArXiv retriever
- **CLI Mode** — Run `main.py` for a terminal-based Q&A session on a pre-indexed document

## 🏗️ How It Works

```
PDF Upload
    ↓
PyPDFLoader  →  RecursiveCharacterTextSplitter
                (chunk_size=1000, chunk_overlap=200)
    ↓
HuggingFaceEmbeddings (all-MiniLM-L6-v2)
    ↓
ChromaDB Vector Store
    ↓
User Question  →  MMR Retriever (k=3, fetch_k=10)
    ↓
Top-3 Relevant Chunks  →  Prompt Template
    ↓
Mistral AI (mistral-small-latest)
    ↓
Answer  +  Retrieved Context View
```

## 🚀 Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/harshit8204/RAG-book-assistant.git
cd RAG-book-assistant/RAG
```

### 2. Create a Virtual Environment

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Set Up Environment Variables

Create a `.env` file inside the `RAG/` folder:

```env
MISTRAL_API_KEY=your_mistral_api_key_here
```

## 🖥️ Usage

### Streamlit Web App

```bash
streamlit run app.py
```

1. Open `http://localhost:8501`
2. Upload any PDF — automatically split into chunks and indexed into ChromaDB
3. Type a question in the input box
4. View the answer and expand **"Retrieved Context"** to see the exact source passages used

### CLI Mode

First, index your PDF into ChromaDB:

```bash
python create_database.py
```

Then run an interactive Q&A session in the terminal:

```bash
python main.py
# Type your question and press Enter
# Type 0 to exit
```

## ⚙️ Key Implementation Details

**MMR Retrieval** — Instead of plain similarity search which returns redundant chunks, MMR balances relevance with diversity:

```python
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 3, "fetch_k": 10, "lambda_mult": 0.5}
)
```

**Strict Context Grounding** — The system prompt forces the LLM to answer only from retrieved context:

```
Use ONLY the provided context to answer the question.
If the answer is not present in the context,
say: "I could not find the answer in the document."
```

**Chunk Settings** — 1000-character chunks with 200-character overlap to preserve context across boundaries.
